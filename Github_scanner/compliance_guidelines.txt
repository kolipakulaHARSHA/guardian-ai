# Sample Compliance Guidelines for Guardian AI
# One guideline per line
# Use these with: python cli.py compliance <repo> --guidelines-file compliance_guidelines.txt

# Documentation Requirements
The project must have a README file with clear installation instructions
The project must have API documentation or user guide
The project must have a CONTRIBUTING guide for new contributors
The project must have inline code comments explaining complex logic

# Licensing and Legal
The project must have a LICENSE file
The project must not contain proprietary or copyrighted code without permission
All dependencies must have compatible licenses

# Security Requirements
The project must have a SECURITY.md file or security policy
Sensitive data such as API keys or passwords must not be hardcoded
The project must use HTTPS for all external API calls
Authentication mechanisms must be properly implemented

# Code Quality
The project must include automated tests
The project must follow consistent code style guidelines
The code must handle errors gracefully with try-catch blocks
The project must not contain unused or dead code

# Dependency Management
All dependencies must be explicitly listed in requirements.txt or package.json
Dependencies should be pinned to specific versions
The project must not use deprecated dependencies
Security vulnerabilities in dependencies must be addressed

# Version Control
The project must have a meaningful .gitignore file
Commit messages should be descriptive and follow conventions
The project must not contain sensitive files in version control
Binary files should be minimized in the repository

# Configuration
Configuration should be separated from code
Environment variables should be used for sensitive configuration
The project must have a .env.example file showing required variables

# DevOps and CI/CD
The project should have continuous integration configured
The project should have automated deployment scripts or documentation
Build and deployment processes must be documented
